# Port Number ranges

Port number are assigned by the Internet Assigned Numbers Authority (IANA) (or by Internet Corporation for Assigned Names and Numbers (ICANN)

  • 0 - 1023 (classic/known_ports)
  • 1024 - 49151 ( Registered ports )
  • 49152 - 65536 ( Dynamic ports )

Which Protocol uses TCP or UDP

  • HTTP:80
    • TCP, because connection oriented
  • FTP: 20,21
    • TCP, file transfer needs to maintain order of segments.
  • DNS:53
    • UDP, Fast resolution of domain name.
  • TFTP:23
    • TCP, Trivial is meant for large files transfer, smaller header in UDP
  • DHCP:67,68
    • UDP, Fast IP address allotment.
  • SSH:22
    • TCP, Reliable connection

# TCP Window Size

  • Window Size 16bits in TCP header: Amount of the data in bytes that can flow, flow control is defined by window size.
  • MSS (Max Segment Size): 1460 bytes

    • So, if 16 bits of Window Size represent 12000 bytes, then total of 12000 bytes of data can be received by buffers at the receiving side. If Segment size is 1200 bytes, then there can be 10 segments overall in the handshake process.

  • MTU Max transmission unit: is the largest packet or frame size, that can be sent in a packet- or frame-based network such as the internet. The maximum MTU of an interface will depend on the hardware platform, but the IEEE 802.3 standards require a minimum MTU of 1500 bytes.

    • As you can see in the figure above, the MTU is built from payload (also referred as data) and the TCP and the IP header, 20 bytes each. The total value of the IP and the TCP header is 40 bytes and mandatory for each packet, which leaves us 1460 bytes for our data.

  • Window Scale factor: n
  • Window size: With Scale factor Window size is Window Size* 2(raised to power n)
